# Implementation Plan: Family Unification

**Branch**: `003-family-unification` | **Date**: 2026-04-13 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/003-family-unification/spec.md`

---

## Summary

Consolidate the separate `parents` and `kids` database tables into a unified `members` table, connected to `families` through a `family_members` join table. Replace all related foreign keys across `sessions`, `chores`, `chore_completions`, and `prize_redemptions`. Rename the `/admin/kids` route to `/admin/family`. Update the header member switcher to show all active members (pills ≤5, dropdown ≥6). All changes are implemented using the existing stack: TypeScript + SvelteKit 2 + Drizzle ORM + SQLite + Skeleton v4.

## Implementation Phases

### Phase 1 — Schema & Migration

**Goal**: New Drizzle schema is applied, data is migrated, app boots with new tables.

1. Update `src/lib/server/db/schema.ts`:
   - Add `members` table (replace `parents` + `kids`)
   - Add `familyMembers` join table
   - Update `sessions`: `userId` → `memberId`, `userRole` → `memberRole`, enum `'parent'|'kid'` → `'admin'|'member'`
   - Update `chores`: `assignedKidId` → `assignedMemberId` (FK → `members`)
   - Update `choreCompletions`: `kidId` → `memberId` (FK → `members`); rename unique index
   - Update `prizeRedemptions`: `kidId` → `memberId` (FK → `members`)
   - Remove `parents` and `kids` table exports

2. Generate SQL migration:
   ```bash
   npm run db:generate
   ```
   Review generated `drizzle/0001_*.sql` to confirm data-migration steps are included, then hand-edit to add the INSERT...SELECT statements (Drizzle kit cannot auto-generate data migrations).

3. Apply migration:
   ```bash
   npm run db:migrate
   ```

**Exit criteria**: `npm run db:migrate` completes without error. App starts and does not crash on startup.

---

### Phase 2 — Auth & Session Layer

**Goal**: Login, signup, session validation, and hooks all work with the new `members` table.

1. Update `src/lib/server/auth.ts`:
   - `SessionPayload`: `userId`/`userRole` → `memberId`/`memberRole`
   - `createSession`: use new column names
   - `validateSession`: returns new shape
   - Remove `getParentById`, `getKidById`
   - Add `getMemberById(id)` — queries `members` table
   - Add `getMemberByEmail(email)` — for admin login
   - Add `getMemberByDisplayName(displayName)` — for member login

2. Update `src/app.d.ts`:
   - `userId` → `memberId`, `userRole` → `memberRole`, `avatarEmoji` non-optional

3. Update `src/hooks.server.ts`:
   - Import `getMemberById` only
   - Resolve user from `session.memberId`
   - Build `event.locals.session` with new shape

4. Update `src/routes/(auth)/login/+page.server.ts`:
   - Admin path: look up `members` by `email`, verify `password_hash`
   - Member path: look up `members` by `displayName`, verify `pin` — remove family code step
   - Create session with `memberId`/`memberRole`; resolve `familyId` from `family_members`

5. Update `src/routes/(auth)/signup/+page.server.ts`:
   - Insert into `members` (first admin) + `family_members` instead of `parents`
   - `userRole: 'admin'` in session

**Exit criteria**: Admin can log in, member can log in with display name + PIN, no TypeScript errors.

---

### Phase 3 — Admin Family Page

**Goal**: `/admin/family` works fully; old `/admin/kids` redirects.

1. Create `src/routes/(app)/admin/family/+page.server.ts`:
   - `load`: query all members in the family via `family_members` join; include coin balances
   - `create` action: insert into `members` + `family_members`; validate role-specific fields (email/pin)
   - `update` action: update `members` + `family_members.role`; enforce last-admin lockout
   - `deactivate` action: set `members.is_active = false`; enforce last-admin lockout

2. Create `src/routes/(app)/admin/family/+page.svelte`:
   - Single member list showing name, avatar emoji, role badge, active status
   - Add member form (role selector toggles email/pin fields)
   - Edit member inline or in modal
   - Deactivate button with confirmation

3. Create `src/routes/(app)/admin/kids/+page.server.ts` (redirect shim):
   ```ts
   import { redirect } from '@sveltejs/kit';
   export const load = () => redirect(301, '/admin/family');
   ```
   (Keep the old `+page.svelte` file if needed for redirect, or rely on the load redirect alone)

4. Update all NavTabs/links that reference `/admin/kids` → `/admin/family`

**Exit criteria**: Navigating to `/admin/family` shows all members. Navigating to `/admin/kids` redirects. CRUD operations succeed.

---

### Phase 4 — Layout & Member Switcher

**Goal**: All pages load with `members` instead of `kids`; switcher works correctly.

1. Update `src/routes/(app)/+layout.server.ts`:
   - Query active members through `family_members` join (not directly from `kids`)
   - Rename `activeKids`/`kidsWithBalances` → `activeMembers`/`membersWithBalances`
   - URL param: `?kid=` → `?member=`
   - Session check: `session.memberRole === 'member'` (was `'kid'`)

2. Update `src/routes/(app)/+layout.svelte`:
   - Pass `members` and `activeMemberId` props down
   - Update slot props to `Header`

3. Create `src/lib/components/MemberSwitcher.svelte`:
   - Props: `members: Array<{id, displayName, avatarEmoji, coinBalance}>`, `activeMemberId: string | null`
   - `≤1` member → renders nothing
   - `≤5` members → pill buttons (Skeleton `btn`, `preset-filled-primary-500` for active)
   - `≥6` members → `<select>` dropdown; `on:change` navigates to `?member=<id>` with `goto()`
   - URL param helper uses `?member=` instead of `?kid=`

4. Update `src/lib/components/Header.svelte`:
   - Import `MemberSwitcher` instead of `KidSwitcher`
   - Prop `kids` → `members`, `activeKidId` → `activeMemberId`
   - Role check: `user.role === 'admin'` (was `'parent'`)

5. Delete `src/lib/components/KidSwitcher.svelte`

**Exit criteria**: Header shows correct member switcher in pill and dropdown modes. Active member highlights correctly. Page data reflects selected member.

---

### Phase 5 — Other App Pages

**Goal**: Chores, leaderboard, prizes, and activity pages work with unified member model.

1. `src/routes/(app)/chores/+page.server.ts`
   - Read `activeMemberId` from URL param `?member=`

2. `src/routes/(app)/admin/chores/+page.server.ts` + `+page.svelte`
   - Assignee picker: load active members from `family_members` join, not `kids` table
   - `assignedKidId` → `assignedMemberId` in form field and DB query

3. `src/routes/(app)/admin/activity/+page.server.ts`
   - Joins with `members` instead of `kids`; `kid_id` → `member_id` in query aliases

4. `src/routes/(app)/leaderboard/+page.server.ts`
   - Show all active `family_members` ranked by coin balance (completions - redemptions)

5. `src/routes/(app)/prizes/+page.server.ts`
   - `prizeRedemptions.memberId` in queries

**Exit criteria**: All admin and user-facing pages load without errors. Chore assignment works for admin-role members.

---

### Phase 6 — Tests & Seed

**Goal**: All automated tests pass; seed produces valid data.

1. Update `src/lib/server/db/seed.ts`:
   - Use `members` + `family_members` schema
   - Create 1 admin + 2 member accounts
   - Log display names and PINs (not family code) at end of seed output

2. Update `tests/integration/admin-kids.test.ts` → `tests/integration/admin-family.test.ts`:
   - Use `members` + `family_members` helpers in `seedFamily()`
   - Test `create`, `update`, `deactivate` actions against new route

3. Update remaining integration tests:
   - `chore-completion.test.ts`: `kidId` → `memberId`
   - `leaderboard.test.ts`: seed members via `members` + `family_members`
   - `prize-redemption.test.ts`: `kidId` → `memberId`

4. Review all e2e tests for hardcoded `parent`/`kid` role checks or `/admin/kids` URLs

5. Run full test suite: `npm test`

**Exit criteria**: `npm test` passes with zero failures. `npm run lint` passes.
