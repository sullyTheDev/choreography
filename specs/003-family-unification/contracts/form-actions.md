# Contracts: Family Unification

**Feature**: `003-family-unification`  
**Date**: 2026-04-13

This document covers the server-side form action contracts and session/load data shapes that change in this feature. SvelteKit form actions serve as the primary interface between browser and server.

---

## Changed Form Actions

### `POST /login` — `action: login`

File: `src/routes/(auth)/login/+page.server.ts`

#### Admin login (email + password)

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `role` | `"admin"` | yes | Selects login path |
| `email` | string | yes | Looked up in `members.email` |
| `password` | string | yes | Compared with `members.password_hash` |

Success → redirect `302 /admin/chores`  
Failure → `fail(401, { error: string })`

#### Member login (display name + PIN)

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `role` | `"member"` | yes | Selects login path |
| `displayName` | string | yes | Looked up in `members.display_name` (globally unique) |
| `pin` | string | yes | Compared with `members.pin` (bcrypt) |

Success → redirect `302 /chores`  
Failure → `fail(401, { error: string })`

**Breaking change from previous**: The member login path previously accepted `familyCode + pin`. It now accepts `displayName + pin` instead (family code step removed).

---

### `POST /signup` — `action: default`

File: `src/routes/(auth)/signup/+page.server.ts`

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `email` | string | yes | Must match `email` format; unique in `members` |
| `password` | string | yes | Min 8 chars |
| `displayName` | string | yes | Globally unique in `members` |
| `familyName` | string | yes | Creates the `families` row |

Creates: `families` (1), `members` (1), `family_members` (1 with `role='admin'`), `sessions` (1).

---

### `POST /admin/family` — `action: create`

File: `src/routes/(app)/admin/family/+page.server.ts`

Creates a new family member of any role.

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `displayName` | string | yes | Globally unique |
| `avatarEmoji` | string | yes | Single emoji character |
| `role` | `"admin" \| "member"` | yes | Stored in `family_members.role` |
| `email` | string | if role=admin | Unique, valid email format |
| `password` | string | if role=admin | Min 8 chars |
| `pin` | string | if role=member | 4–6 digits |

Success → `200 { success: true }`  
Failure → `fail(400, { error: string })`

---

### `POST /admin/family` — `action: update`

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `id` | string | yes | `members.id` |
| `displayName` | string | yes | New display name |
| `avatarEmoji` | string | yes | New emoji |
| `role` | `"admin" \| "member"` | yes | New role for this family |
| `email` | string | conditional | Required when role=admin |
| `pin` | string | conditional | Required when role=member |

Business rule: blocked if `role` → `'member'` and this member is the last `admin` in the family.

---

### `POST /admin/family` — `action: deactivate`

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `id` | string | yes | `members.id` to deactivate |

Sets `members.is_active = false`.  
**Blocked** if member is the last active `admin` in this family → `fail(400, { error: string })`.

## Load Data Shape Changes

### `src/routes/(app)/+layout.server.ts` — `LayoutData`

```ts
// Before
{
  user: { id, displayName, role: 'parent' | 'kid', avatarEmoji?: string },
  family: { id, name, familyCode, leaderboardResetDay },
  kids: Array<{ id, displayName, avatarEmoji, coinBalance }>,
  activeKidId: string | null
}

// After
{
  user: { id, displayName, role: 'admin' | 'member', avatarEmoji: string },
  family: { id, name, familyCode, leaderboardResetDay },
  members: Array<{ id, displayName, avatarEmoji, coinBalance }>,
  activeMemberId: string | null
}
```

### `src/routes/(app)/admin/family/+page.server.ts` — `PageData`

```ts
{
  members: Array<{
    id: string,
    displayName: string,
    avatarEmoji: string,
    role: 'admin' | 'member',
    isActive: boolean,
    coinBalance: number
  }>
}
```

---

## Session Cookie — No change

The session cookie name (`SESSION_COOKIE_NAME`) and options are unchanged.  
The DB-side `sessions` table schema changes (new columns), but the cookie value (UUID) and transport are identical.

---

## URL Parameter Change

| Context | Before | After |
|---------|--------|-------|
| Active member URL param | `?kid=<id>` | `?member=<id>` |
| All links generated by `MemberSwitcher` | `?kid=...` | `?member=...` |
| Layout server reads | `url.searchParams.get('kid')` | `url.searchParams.get('member')` |
